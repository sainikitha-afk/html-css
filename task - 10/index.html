// 01 · HTML

Semantic
HTML.

Structure your content with meaning. Search engines and screen readers both thank you.

HTML
Structure

Semantic Elements

<header>  
<nav>     
<main>    
<article> 
<section> 
<aside>   
<footer>

Semantic tags tell the browser what your content is, not just how it looks. Always prefer them over generic <div>s.

Forms

Input Elements

<form action="/submit">
  <label for="name">Name</label>
  <input type="text"
         id="name"
         required/>
  <button type="submit">
    Submit
  </button>
</form>

Always pair <label> with its input using matching for / id for full accessibility.

Meta & Head

Document Head

<head>
  <meta charset="UTF-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <meta name="description" content="Page description for SEO"/>
  <title>Page Title</title>
  <link rel="stylesheet" href="styles.css"/>
</head>

The viewport meta tag is the single most important line for responsive design. Never skip it.

Media

Images & Links

<img
  src="photo.jpg"
  alt="Descriptive text"
  loading="lazy"
  width="800"
  height="600"
/>

<a href="https://..."
   target="_blank"
   rel="noopener">
  Link text
</a>

alt text is mandatory. loading="lazy" defers off-screen images for performance.

Lists

List Types

/* Unordered */
<ul>
  <li>Item one</li>
  <li>Item two</li>
</ul>

/* Ordered */
<ol type="A">
  <li>First</li>
  <li>Second</li>
</ol>

Use <ul> for unordered nav items, <ol> for steps or rankings.

Variables

Custom Properties

:root {
  --color-primary: #ff6b35;
  --font-size-lg: 1.25rem;
  --spacing-md: 1.5rem;
  --radius: 8px;
}

.button {
  background: var(--color-primary);
  padding: var(--spacing-md);
  border-radius: var(--radius);
}

Define once in :root, reuse everywhere. Changing a theme is one line.

Pseudo-classes

State Selectors

a:focus     { outline: 2px solid blue; }
a:visited   { color: purple; }
input:checked{ accent-color: lime; }
p:first-child{ font-weight: bold; }
li:nth-child(2n) {
  background: #f5f5f5;
}

  opacity: 1;
}
Transitions

Smooth Changes

.card {
  transition:
    transform 0.3s ease,
    box-shadow 0.3s ease,
    opacity 0.2s linear;
}

.card:hover {
  transform: translateY(-8px);
  box-shadow: 0 20px 40px
    rgba(0,0,0,0.15);
}

Always transition only transform and opacity — they're GPU accelerated.

Animations

@keyframes

@keyframes fadeSlideUp {
  from {
    opacity: 0;
    transform: translateY(30px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.hero-text {
  animation: fadeSlideUp 0.8s ease 0.2s both;
}   /* name | duration | easing | delay | fill-mode */

both as fill-mode applies the from state before playback and the to state after — prevents flicker.

Flexbox

Two layout systems. Grid for the macro, Flex for the micro. Learn both cold.

Layout
Flexbox

Flex Container

  display: flex;
  flex-direction: row;      /* row | column */
  justify-content: space-between; /* main axis */
  align-items: center;     /* cross axis */
  flex-wrap: wrap;
  gap: 1rem;

  flex: 1 1 200px; /* grow shrink basis */
}
CSS Grid

Grid Container

Header
Sidebar
Main
Footer
.grid {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: auto 1fr auto;
  gap: 1rem;
}
.header { grid-column: 1 / -1; }
.footer { grid-column: 1 / -1; }

/* Responsive shorthand */
grid-template-columns:
  repeat(auto-fit, minmax(280px, 1fr));
Responsive

Media Queries

/* Mobile first approach */
.grid {
  grid-template-columns: 1fr;
}

@media (min-width: 768px) {
  .grid {
    grid-template-columns:
      repeat(2, 1fr);
  }
}

@media (min-width: 1024px) {
  .grid {
    grid-template-columns:
      repeat(3, 1fr);
  }
}
Modern CSS

clamp() & fluid type

/* clamp(min, preferred, max) */
h1 {
  font-size: clamp(2rem, 5vw, 5rem);
}
.container {
  padding:
    clamp(1rem, 3vw, 3rem);
  max-width: min(90%, 1200px);
  margin: 0 auto;
}
/* No media query needed! */

clamp() replaces dozens of media query breakpoints with fluid scaling.

Pattern

:target Modal

/* HTML */
<a href="#modal">Open</a>
<div id="modal" class="modal">
  <a href="#" class="close">✕</a>
</div>

/* CSS */
.modal {
  opacity: 0;
  pointer-events: none;
  transition: opacity .3s;
}
.modal:target {
  opacity: 1;
  pointer-events: all;
}

Zero JavaScript. Pure CSS modal using URL hash and :target.

Pattern

Checkbox Accordion

Content + Padding + Border + Margin. Always use box-sizing: border-box.
Grid = 2D layout. Flex = 1D (row or column). Use both together.
Pattern

Dropdown Menu

.dropdown { position: relative; }

.dropdown-menu {
  position: absolute;
  top: 100%;
  opacity: 0;
  visibility: hidden;
  transform: translateY(-8px);
  transition: opacity .25s, transform .25s,
    visibility .25s;
}

.dropdown:hover .dropdown-menu,
.dropdown:focus-within .dropdown-menu {
  opacity: 1;
  visibility: visible;
  transform: translateY(0);
}

:focus-within makes keyboard navigation work — the dropdown stays open when focus is inside it.
